Process

  • I started with the class demos (pattern + grid + simple motion) and then explored how to build my own grid system.
  • I created a small g(n) grid helper so I could place shapes by cells instead of pixels, which made everything easier to control and align.
  • I kept the randomized black rectangles generated inside setup(), so each refresh creates a new layout.
  • For motion, I reused the bounce logic from class for the big red square (same idea as the moving circle).
  • I added a slower, calmer motion for the yellow rectangle using sin(frameCount * freq), which gave the piece a more subtle rhythm.

Challenges

  • Grid scale & density: My early grids were too tight visually. I switched to larger cells to make the piece feel more “block print” and closer to a Mondrian layout.
  • Overlap control: I had to prevent random black rectangles from spawning on top of the moving red square or the blue bar. This required extra checks for collisions.
  • Motion pacing: I tuned the sway speed and amplitude so the yellow block feels smooth instead of jittery. Getting that “calm motion” feeling took a few iterations.

What I’d try next

  • Swap out different guide-line layouts to study balance and negative space in different compositions.
  • Light “paper” texture pass (very subtle) to test print vibes.
  • next step would be adding a subtle paper texture or a Mondrian reference image to reinforce the printmaking or painting feeling.
