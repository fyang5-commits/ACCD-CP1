Mondrian-Style Pattern, Rhythm & Movement

p5.js study — grid composition + simple motion

grid() random() bounce sin() sway

Process (quick notes)

  • Started from class references: pattern + grid with simple motion.
  • Built a grid helper g(n) to place things by “cells,” not pixels.
  • Kept static black blocks randomized in setup() (new layout on refresh).
  • Used bounce (vx/vy) for the big red square — same logic as the moving circle demo.
  • Added one slow sway rectangle (yellow) using sin(frameCount * freq).
  • Palette sticks to red/blue/yellow/black/white for that abstract print vibe.

Challenges

  • Grid scale & density: early grid was too tight; switched to bigger cells for blocky “print” feel.
  • Overlap control: avoided placing random blacks on top of the red square / color bar.
  • Motion pacing: tuned freq and amp so the yellow sway feels calm, not jittery.
  • Teacher-style code: replaced raw pixel math with a simple g(n) function + guide lines array.

What I’d try next

  • Swap guide lines per scene to study different balance/negative space.
  • Light “paper” texture pass (very subtle) to test print vibes — optional.
  • Param knobs in UI (grid size, sway speed) for quick iterations in class.
