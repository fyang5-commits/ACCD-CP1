Process & Learning

For this project, I followed the Donut tutorial to get hands-on practice with Blender. My main goal wasn’t just to finish the donut, but to get comfortable with the basics, hotkeys, modifiers, materials, lighting, and even a bit of geometry nodes.

It definitely wasn’t a smooth ride. I got lost a few times because my Blender version was different from the one in the tutorial, so some tools were renamed or hidden in different places. A few steps didn’t work the same way on my screen, which made me pause and troubleshoot on my own.

In my first render, the sprinkle shapes came out tiny, like microscopic. I had to readjust the scale manually, and while it improved a little, it still didn’t look quite right. In the second render, I ran into a rotation issue, the pivot point of my donut somehow ended up far away, so rotating it made the whole thing spin weirdly. By the third render, I finally figured it out and fixed the problem.

Even though these mistakes were frustrating, they actually helped me understand Blender’s logic better. I started to see how object origins, scaling, and geometry nodes all connect. Now when something goes wrong, I don’t panic, I look for the cause and learn from it.

What I learned: This project taught me that making mistakes is part of the learning process. Every failed render brought me one step closer to understanding how Blender really works. Next time, I want to experiment more confidently and start tweaking materials and lighting on my own instead of just following the tutorial.

Notes & Hotkeys

  • Shift + A — Add new object
  • G — Move object
  • R — Rotate object
  • S — Scale object
  • M — Mask
  • Tab — Switch between Edit and Object mode
  • Shift + D — Duplicate object
